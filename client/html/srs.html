1. Introducere

1.1 Scop

Scopul acestui document este de a specifica cerințele pentru sistemul de management al abrevierilor APlace. Sistemul este conceput pentru a oferi utilizatorilor o platformă completă pentru crearea, gestionarea și partajarea abrevierilor din diverse domenii și limbi.

Principalul obiectiv este centralizarea managementului abrevierilor, oferind funcționalități avansate precum statistici în timp real, vizualizare colaborativă, export de date și generare automată de fluxuri RSS pentru conținutul popular.

1.2 Domeniu de aplicare

Sistemul APlace VA oferi:

  • Autentificare și autorizare cu control al accesului pe bază de roluri
  • Operațiuni CRUD complete pentru gestionarea abrevierilor
  • Căutare avansată și filtrare
  • Funcționalitate de export date în mai multe formate (HTML, Markdown)
  • Statistici în timp real și grafice interactive
  • Monitorizare vizualizări și sistem de clasament pe baza popularității
  • Generare de fluxuri RSS pentru cele mai vizualizate abrevieri
  • Interfață web responsivă pentru desktop și mobil

Sistemul APlace NU VA oferi:

  • Gestionarea de imagini sau conținut multimedia
  • Servicii de notificare prin email
  • Procesare de plăți
  • Funcționalități de chat sau mesagerie în timp real
  • Integrare cu rețele sociale terțe

2. Descriere generală

2.1 Perspectiva produsului

APlace este o aplicație web independentă care funcționează fără dependențe externe. Sistemul urmează o arhitectură pe trei niveluri:

  • Prezentare: Frontend HTML5, CSS3 și JavaScript
  • Aplicație: Server Node.js cu API RESTful
  • Date: Bază de date SQLite cu mecanisme de cache

Sistemul este proiectat să fie autonom și poate fi instalat pe orice mediu ce suportă Node.js.

2.2 Funcționalitățile produsului

Principalele funcționalități oferite de sistemul APlace sunt:

Autentificare și autorizare

  • Înregistrare și autentificare utilizatori
  • Gestionare sesiune pe bază de JWT
  • Control al accesului pe roluri (Guest, Utilizator, Admin)
  • Funcție de recuperare parolă

Management abrevieri

  • Crearea de abrevieri cu metadate
  • Vizualizarea și căutarea abrevierilor existente
  • Editarea detaliilor unei abrevieri
  • Ștergerea abrevierilor (cu permisiuni corespunzătoare)

Căutare și descoperire

  • Căutare text în abrevieri și explicații
  • Filtrare după limbă, domeniu și autor
  • Paginare pentru rezultate mari
  • Sugestii în timp real la căutare

Analiză și statistici

  • Monitorizare vizualizări pentru măsurarea popularității
  • Grafice interactive și vizualizări statistice
  • Top 10 cele mai vizualizate abrevieri
  • Actualizare statistici în timp real

Export și partajare date

  • Export în format HTML
  • Export în format Markdown
  • Generare flux RSS pentru conținutul popular
  • Formatare print-friendly

2.3 Caracteristici utilizatori

Utilizatori guest

  • Acces: Doar vizualizare pentru abrevierile publice
  • Competențe tehnice: Abilități de bază de navigare web
  • Scop principal: Răsfoire și vizualizare abrevieri

Utilizatori autentificați

  • Acces: Operațiuni CRUD complete pe propriile abrevieri
  • Competențe tehnice: Cunoștințe informatice intermediare
  • Scop principal: Gestionarea colecțiilor proprii de abrevieri

Administratori

  • Acces: Administrare completă a sistemului
  • Competențe tehnice: Cunoștințe tehnice avansate
  • Scop principal: Management sistem și administrare utilizatori

2.4 Constrângeri

Constrângeri tehnice

  • Necesită browser modern cu JavaScript activat
  • Serverul rulează în mediu Node.js
  • Persistența datelor în SQLite
  • Conectivitate la rețea pentru funcții în timp real

Constrângeri operaționale

  • Arhitectură single-server
  • Stocare locală pe sistemul de fișiere
  • Disponibilitate port 8080 pentru server web

Constrângeri legislative

  • Respectarea standardelor de accesibilitate web
  • Respectarea normelor de confidențialitate și protecție a datelor
  • Respectarea licențierii open-source

2.5 Presupuneri și dependențe

Presupuneri

  • Utilizatorii au cunoștințe de bază privind utilizarea aplicațiilor web
  • Conexiune stabilă la internet pentru performanță optimă
  • Browser modern cu suport HTML5 și CSS3
  • JavaScript activat în browser

Dependențe

  • Node.js (versiunea 16 sau mai nouă)
  • Motorul de baze de date SQLite
  • Librăria Chart.js pentru vizualizări
  • Librăria Mermaid.js pentru generare diagrame

3.2 Cerințe de performanță

Cerințe de timp de răspuns

  • Timpul de încărcare al paginilor nu va depăși 3 secunde în condiții normale
  • Operațiile CRUD se vor finaliza în maxim 2 secunde
  • Căutările vor returna rezultate în maxim 1 secundă
  • Operațiile de export vor fi finalizate în maxim 10 secunde

Cerințe de debit

  • Sistemul va suporta minim 10 utilizatori concurenți
  • Fiecare cont poate avea până la 100 de abrevieri
  • Sistemul va procesa 50 de cereri de căutare pe minut

Cerințe de disponibilitate

  • Sistemul va asigura o disponibilitate de 99% în timpul programului de lucru
  • Ferestrele planificate de mentenanță nu vor depăși 2 ore

3.3 Constrângeri de proiectare

Constrângeri tehnologice

  • Frontend-ul va fi implementat folosind HTML5, CSS3 și JavaScript simplu
  • Backend-ul va fi implementat în Node.js
  • Baza de date va folosi SQLite pentru persistența datelor
  • Nu se vor folosi baze de date externe

Constrângeri de interfață

  • Interfața va fi responsivă și compatibilă cu dispozitive mobile
  • Sistemul va funcționa pe browserele Chrome, Firefox, Safari și Edge
  • Rezoluția minimă suportată va fi de 320px lățime

3.4 Atributele sistemului software

Securitate

  • Parolele utilizatorilor vor fi criptate cu bcrypt
  • Gestionarea sesiunilor se va face prin JWT
  • Validarea inputului va preveni atacurile de tip SQL injection
  • Se va implementa protecție împotriva atacurilor XSS

Fiabilitate

  • Integritatea datelor va fi asigurată prin managementul tranzacțiilor
  • Sistemul va gestiona și înregistra erorile într-un mod grațios
  • Vor exista mecanisme automate de backup

Ușurință în utilizare

  • Interfața va urma principii moderne de design web
  • Navigarea va fi intuitivă și consecventă
  • Mesajele de eroare vor fi clare și utile
  • Sistemul va furniza feedback vizibil la orice acțiune

Mentenenabilitate

  • Codul va respecta principiile de arhitectură modulară
  • Documentația va fi menținută pentru toate componentele sistemului
  • Logurile vor facilita depanarea și monitorizarea sistemului

4.2 Etapele dezvoltării

Sistemul APlace a fost dezvoltat printr-o abordare sistematică, urmând principii agile și implementare incrementală a funcționalităților.

✅ Etapa 1: Setup fundație (finalizată)

  • Infrastructură: Configurare server Node.js cu rutare Express-like
  • Bază de date: Proiectare și implementare schemă SQLite
  • Autentificare: Sistem JWT-based pentru autentificare
  • CRUD de bază: Operații principale pentru abrevieri
  • Structură frontend: Șabloane HTML și stilizare CSS de bază

✅ Etapa 2: Funcționalități de bază (finalizată)

  • Interfață utilizator: Dashboard responsiv și interfață de management
  • Căutare & filtrare: Căutare avansată cu filtre multiple
  • Export date: Export în format HTML și Markdown
  • Statistici: Analytics de bază cu Chart.js
  • Roluri utilizator: Guest, Utilizator, Admin

✅ Etapa 3: Funcționalități avansate (finalizată)

  • Monitorizare vizualizări: Sistem de popularitate în timp real
  • Top abrevieri: Top 10 dinamic pentru abrevierile vizualizate
  • Integrare RSS: Generare automată de fluxuri RSS
  • Modale interactive: Vizualizări detaliate pentru UX îmbunătățit
  • Actualizări live: Statistici și clasament cu refresh automat

✅ Etapa 4: Producție și optimizare (finalizată)

  • Mașină virtuală: Deploy pe VM cu Ubuntu
  • Servire statică: Integrare fișiere statice
  • Configurare port: Acces rețea cu port forwarding
  • Optimizare performanță: Cache la nivel de DB și mecanisme de sincronizare
  • Documentație completă: Sistem complet documentat

🎯 Rezumat implementare

Durată dezvoltare: 4 etape finalizate
Arhitectură: Trei niveluri, RESTful API
Capacitate date: Suport nelimitat pentru abrevieri
Capacitate utilizatori: Multi-utilizator, control pe roluri
Status implementare: Gata de producție pe infrastructură VM
